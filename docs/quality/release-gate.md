# Release Review + QA Gate

This gate is the default pre-handoff check for changes that affect production behavior, public pages, translation, pricing, billing, or the merchant console.

## Command

```powershell
$env:DATABASE_URL = "postgresql://nested_relay:change-me@localhost:5432/nested_relay?schema=public"
npm run qa:release-gate
```

If `DATABASE_URL` is omitted, the gate uses the local development database above. By default the gate builds the project, starts managed API/Web services on free local ports, runs all checks against those managed services, then shuts them down. Set `RELEASE_GATE_USE_EXISTING_SERVICES=true` only when you intentionally want to test already-running services from `API_BASE_URL` and `WEB_BASE_URL`. Override `CHROME_PATH` if Chrome is installed outside the default location.

## Prerequisites

- The database in `DATABASE_URL` has current migrations applied.
- Google Chrome is installed for screenshot and console QA.
- Ports near `3027` and `3028` are available, or override `RELEASE_GATE_API_PORT` / `RELEASE_GATE_WEB_PORT`.
