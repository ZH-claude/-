# T23 M07 上线决策报告

## 结论

结论：暂缓生产上线；可继续受控内测。

当前本地能力已经覆盖翻译产品化、公开内容本地化、review + QA 门禁、运营内容发布工作流、风控成本保护、VibeCoding 套餐、Token 排行榜、手机号登录/找回密码、商家端保存档案、固定上游分配、1000 用户本地并发基线和运维 dry-run 门槛。

但当前仓库仍缺少真实云服务器、正式域名/DNS、生产 `.env`、真实上游 Key、真实 smoke 账号和真实通知渠道。生产 strict smoke、真实生产账单核对、真实恢复演练、真实支付回调验签和外部监控接入尚未完成，不能开放给客户作为正式生产服务。

## P0/P1 阻塞项

| 阻塞项 | 影响 | 解除条件 |
| --- | --- | --- |
| 无真实云服务器 SSH | 不能执行真实部署和服务器级恢复演练 | 提供可用服务器和 SSH |
| 无正式域名 DNS | 不能签发 HTTPS 证书，不能完成真实公网 strict smoke | 配好 `app` 和 `api` 域名 A/AAAA 记录 |
| 无生产 `.env` | 不能启动生产环境，也不能验证真实密钥、cookie、数据库和 Redis 配置 | 在服务器生成真实密钥和连接配置 |
| 无真实上游 Key | 不能完成真实 chat strict smoke 和生产成本核对 | 在商家端或生产环境配置真实上游 |
| 无真实 smoke 账号 | 不能完成登录、令牌、trace、充值和通知 strict smoke | 准备带余额的真实测试账号 |
| 无真实通知渠道 | 不能完成通知 strict smoke 和余额告警闭环 | 配置真实 Webhook 或邮件渠道 |
| 真实支付回调未验签 | 支付宝/微信等真实支付不能作为正式充值入口 | 接入真实商户参数、证书/密钥和回调验签 |
| 外部监控未配置 | 生产可用性和告警不能闭环 | 接入 Uptime Kuma 或等价外部监控，并验证告警 |

## 未完成能力边界

- 真实上游未接入生产：本地 QA 使用临时上游或配置化上游，不等于真实供应商链路已通过。
- 真实支付未接入生产：当前可验证兑换码、钱包和模拟入账路径，真实支付回调验签仍未完成。
- 真实通知未接入生产：本地通知和 Webhook 逻辑可测，生产通知渠道仍未配置。
- 外部监控未接入生产：服务状态页能显示未配置，不会伪造健康；外部监控和告警仍需配置。
- 真实生产恢复演练未完成：本地 dry-run 只证明脚本不变量，不能替代可丢弃环境里的备份恢复和回滚演练。

## 放行建议

- 允许：继续受控内测，继续通过 `npm run qa:release-gate` 做每次变更的交付门槛。
- 不允许：公开承诺正式生产上线、开放大规模付费客户、宣称真实上游/真实支付/真实通知/真实监控已完成。
- 下一步：补齐服务器、域名、生产 `.env`、真实上游 Key、真实 smoke 账号、真实通知渠道和支付商户参数后，执行生产 strict smoke、生产账单核对、生产安全复核、真实恢复演练，再重新生成上线决策。

Review 结论：当前报告没有把缺失的外部条件写成完成项，结论对非技术负责人是可执行的：现在只能受控内测，不能正式开放给客户。
