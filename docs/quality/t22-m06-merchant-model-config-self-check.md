# T22-M06 商家端模型与上游配置自检记录

日期：2026-06-16

## 结论

- M06 已完成当前 MVP 范围：平台老板在商家端配置上游、模型和分组，客户使用本平台发放的密钥后，可以看到被授权的真实模型。
- 当前 MVP 是单平台老板模式，不是多商家入驻模式。
- “多个商家各自保存不同上游密钥，客户按所属商家转发”尚未实现，已作为商用升级计划写入任务规划，未在本任务中硬改。

## 已完成内容

- 新增商家端“上游/模型”独立页面。
- 商家端可管理真实分组、真实模型价格、真实上游配置、真实上游模型映射和健康检查。
- 上游密钥保存后只展示脱敏预览，明文不会从页面和列表返回。
- 登录页、注册页、首页、用户端和商家端公共品牌已改为中文“中转站控制台”。
- 当前页面可见文案已中文化；真实账号名、模型代号、分组代码等业务数据不强行翻译。

## 真实检查

- 自动检查已通过：临时创建后台账号、普通用户、分组、模型、上游、映射和客户密钥。
- 已验证普通用户不能访问或操作商家端配置。
- 已验证新增模型后，用户端价格页、账户可用模型和模型列表能看到同一份真实配置。
- 已验证上游密钥不明文外泄，并且保存内容不是明文。
- 已验证本机、内网和云元数据地址不能通过健康检查。
- 检查结束后临时用户、钱包、会话、分组、模型、上游、映射、客户密钥、日志记录均清理为 0。

- 本地商家账号进入“上游/模型”页面通过。
- 顶部固定栏和左侧固定栏存在。
- 当前菜单高亮正确。
- 桌面宽度和手机宽度均无横向撑破。

- 页面没有出现旧英文品牌。
- 页面没有暴露密码、客户密钥摘要、上游密钥、连接串等敏感内容。

## 边界说明

- 当前已经满足“你作为平台老板，从别的中转站拿密钥，填到商家端，再提供给你的客户使用”的 MVP 控制台配置能力。
- 当前尚未完成商用版“多商家独立经营、每个商家独立上游密钥、客户归属不同商家”的隔离能力。
- 商用升级需要按计划新增商家归属、客户归属、商家级上游密钥隔离、请求按商家转发和双商家闭环验收，不能在 M06 中临时混入。

## 副手复核

- 已让副手只读复核链路。
- 副手确认：平台客户密钥、上游密钥加密保存、模型映射和转发骨架已经存在。
- 副手同时指出：多商家独立上游密钥隔离尚未完成，不能声称商用多商家闭环已完成。
- 本次采纳该结论，并将多商家能力放入后续商用升级计划。
