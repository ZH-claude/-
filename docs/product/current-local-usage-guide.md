# 当前本地使用说明

更新时间：2026-06-17

## 先看这两个数字

金额按两端分开显示：用户端使用美元，看到 `$10.00` 就是 10 美元；商家端使用人民币，看到 `¥10.00` 就是人民币显示口径。当前只做分端显示，不做自动汇率换算。

| 页面文字 | 它代表什么 | 它不代表什么 |
| --- | --- | --- |
| 客户剩余额度 | 所有客户钱包里还没用完的额度总和 | 不是你的银行卡余额，不是平台利润 |
| 今日客户扣费 | 今天客户调用模型后被扣掉的额度 | 不是上游成本，不是最终利润 |
| 累计客户扣费 | 客户历史调用后累计被扣掉的额度 | 不是已提现收入 |

利润要等后续做“上游成本”和“客户扣费”的对账表后才能准确看。现在这个首页先帮你看客户额度池和客户当天使用情况。

## 商家端怎么用

1. 进入商家首页，先看客户剩余额度、今日客户扣费、活跃用户、上游状态。
2. 打开“上游 API”，填写你从别的中转站拿到的上游地址和密钥，保存后做健康检查。
3. 打开“模型发布”，填写要给客户看的模型名、展示名、输入价格、输出价格和倍率，然后保存。商家端价格按“人民币 / 1K”显示和填写。
4. 如果模型填错了，在“模型列表”里点“修改”，左侧表单会变成修改当前模型；没有绑定上游时可以改模型名，已经绑定上游后只能改展示名、价格、倍率和启停状态。
5. 打开“模型映射”，把客户看到的模型绑定到真实上游模型。只保存模型名不等于客户可见，必须完成这一步，用户端才会显示这个模型。
6. 打开“用户管理”，查看客户、客户额度、累计扣费和账号状态。
7. 打开“充值码”，填写人民币显示金额并生成充值码，客户兑换后会进入客户余额；用户端看到的是美元显示。
8. 打开“请求日志”和“绘图日志”，查看客户真实调用记录。

## 用户端怎么用

1. 普通用户登录后进入个人中心，看自己的客户剩余额度、累计客户扣费、调用次数和可用模型。
2. 需要加额度时，进入余额充值，输入商家发放的充值码。
3. 需要调用接口时，进入“令牌”页面管理自己的调用密钥。
4. 令牌页面上方可以按令牌名称、可用模型、状态查询；点“重置”会清空筛选条件。
5. 点“新增”创建新令牌，只需要填写名称、可选额度、过期时间、可用模型和备注；不想限制额度或过期时间就留空。
6. 表格每一行右侧可以复制、删除、重置密钥、修改。修改只改令牌信息；重置密钥会生成新的完整密钥。
7. 完整密钥只会在新增或重置后显示一次，必须当场复制保存；之后页面只能复制令牌名称和部分预览。
8. 点“多选”后可以勾选多条令牌一起删除；点“导出”会导出当前筛选结果。
9. 调用后，用户可以在日志页面查看真实调用记录。

## 充值和扣费怎么走

1. 商家先生成充值码，商家端按人民币显示和填写。
2. 用户兑换充值码后，用户端按美元显示余额。
3. 用户调用模型时，系统按模型输入价格、输出价格和模型倍率计算扣费，用户端扣费显示为美元。
4. 调用成功才扣费；调用失败不会扣客户余额。
5. 现在默认归属倍率保持为 1，不需要商家或用户手动操作。

## 外部工具怎么接

同一把用户端令牌可以用于多种工具。先在用户端“令牌”页面新增或重置令牌，并立刻复制完整密钥。

现在你的正式域名还没有接入本项目，所以先用本地地址测试。上线后把 `127.0.0.1:3001` 换成你服务器上的正式接口域名即可，推荐先预留成 `https://api.你的域名.com`。

| 工具类型 | 本地地址 | 上线后地址 |
| --- | --- | --- |
| OpenAI 兼容工具 | `http://127.0.0.1:3001/v1` | `https://api.你的域名.com/v1` |
| Claude Code / CCSwitch | `http://127.0.0.1:3001` | `https://api.你的域名.com` |
| Codex / Responses 兼容工具 | `http://127.0.0.1:3001/v1` | `https://api.你的域名.com/v1` |
| OpenCode | 选择 OpenAI 方式填 `/v1` 地址；选择 Claude 方式填不带 `/v1` 的地址 | 同左，换成上线域名 |

当前已经兼容这些入口：

| 入口 | 用途 | 状态 |
| --- | --- | --- |
| `/v1/models` | 模型列表 | 已接入 |
| `/v1/chat/completions` | OpenAI 兼容聊天 | 已接入 |
| `/v1/responses` | Codex / 新版 OpenAI Responses | 已接入 |
| `/v1/messages` | Claude Code / CCSwitch | 已接入 |
| `/v1/messages/count_tokens` | Claude Code 计数检查 | 已接入，本地估算，不调用上游 |

### Claude Code / CCSwitch 本地示例

```powershell
$env:ANTHROPIC_API_KEY="用户端复制的完整令牌"
$env:ANTHROPIC_AUTH_TOKEN="用户端复制的完整令牌"
$env:ANTHROPIC_BASE_URL="http://127.0.0.1:3001"
$env:ANTHROPIC_MODEL="你发布给客户看的模型名"
$env:ANTHROPIC_DEFAULT_SONNET_MODEL="你发布给客户看的模型名"
$env:ANTHROPIC_DEFAULT_OPUS_MODEL="你发布给客户看的模型名"

claude --model "你发布给客户看的模型名"
```

Claude Code 建议把同一把用户端完整令牌同时填到 `ANTHROPIC_API_KEY` 和 `ANTHROPIC_AUTH_TOKEN`。前者走常规密钥字段，后者走授权字段；我们的接口两种都接受。改完变量后必须重新打开一个新的 PowerShell，再启动 Claude Code。

### Codex / OpenAI 兼容本地示例

```powershell
$env:OPENAI_API_KEY="用户端复制的完整令牌"
$env:OPENAI_BASE_URL="http://127.0.0.1:3001/v1"
```

### OpenCode 怎么选

OpenCode 如果选择 OpenAI 兼容方式，就填 `http://127.0.0.1:3001/v1`；如果选择 Claude 兼容方式，就填 `http://127.0.0.1:3001`。两种方式用的都是用户端生成的完整令牌。

上线后只把地址换掉，令牌仍然使用用户端生成的完整令牌。

## 关于“分组”

页面上已经不再让商家或用户操作“分组”。系统内部仍保留默认归属，用来保证已有客户能继续看到可用模型，不影响充值、令牌、价格和调用。

旧的分组页面入口现在会自动回到正确位置：普通用户回个人中心，商家账号回商家首页。
